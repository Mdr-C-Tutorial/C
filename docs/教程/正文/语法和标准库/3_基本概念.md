# 3. 基本概念

本章负责把后续内容反复依赖的术语先统一起来：对象、类型、翻译单元、作用域、行为的分类、执行环境。

::: tip 可暂时跳过本章

如果你现在更想先上手编码，可以先浏览每节标题，再继续后面的语法章节。遇到“为什么是这个结果”或“为什么这段代码不可靠”这类问题时，再回到本章按顺序精读，效果通常更好。

:::

## 1. 本章主线

我们先从 [3.1 注释](/教程/正文/语法和标准库/3_基本概念/3_1_注释.md) 开始，区分“给编译器看的语法”与“给读者看的说明”。随后进入 [3.2 对象](/教程/正文/语法和标准库/3_基本概念/3_2_对象.md)，明确 C 语言里的对象不是面向对象术语，而是可存储数据的区域；它的规则由 [3.2.1 类型](/教程/正文/语法和标准库/3_基本概念/3_2_对象/3_2_1_类型.md) 与 [3.2.2 声明和初始化](/教程/正文/语法和标准库/3_基本概念/3_2_对象/3_2_2_声明和初始化.md) 进一步展开。

然后是名字系统：在 [3.3 标识符](/教程/正文/语法和标准库/3_基本概念/3_3_标识符.md) 中你会看到命名和可见范围的规则，其中 [3.3.1 保留标识符](/教程/正文/语法和标准库/3_基本概念/3_3_标识符/3_3_1_保留标识符.md) 解释哪些名字不应由用户代码占用，[3.3.2 作用域](/教程/正文/语法和标准库/3_基本概念/3_3_标识符/3_3_2_作用域.md) 解释名字在源码中的可见区间。

接着通过 [3.4 翻译单元](/教程/正文/语法和标准库/3_基本概念/3_4_翻译单元.md) 与 [3.5 翻译阶段](/教程/正文/语法和标准库/3_基本概念/3_5_翻译阶段.md) 把“源代码到可执行程序”的路径串起来，避免把预处理、编译和链接混为一层。[3.6 遵从性和行为](/教程/正文/语法和标准库/3_基本概念/3_6_遵从性和行为.md) 会给出标准层面的行为边界，解释实现定义行为、未指定行为与未定义行为之间的区别——这套分类是后续判断“哪些写法可靠、哪些写法不可靠”的前提。最后通过 [3.7 ASCII 码](/教程/正文/语法和标准库/3_基本概念/3_7_ASCII码.md) 与 [3.8 环境](/教程/正文/语法和标准库/3_基本概念/3_8_环境.md) 收束到“字符表示”和“程序运行前提”。
