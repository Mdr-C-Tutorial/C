# 派生类型

派生类型是在已有类型基础上继续构造的新类型族。它们不是“额外语法糖”，而是 C 组织数据与接口的核心工具：数组表达批量同构数据，指针表达间接访问，结构体与联合体表达复合布局，函数类型表达可调用接口。

## 1. 学习顺序

本章按“先内存形状、后接口语义”的顺序展开：先讲数组和指针，再讲结构体与联合体，最后讲原子相关能力。这样安排的目的，是让你在阅读复杂声明时始终能落回同一个问题：这段声明最终描述的对象布局与访问规则是什么。

## 2. 一段组合示例

```c
#include <stddef.h>

typedef struct {
    int *data;
    size_t size;
} int_vector;
```

这里 `int_vector` 同时用到了结构体、指针和基本类型。`data` 指向一段连续整型对象，`size` 描述元素个数。几乎所有工程中的容器抽象都可还原为这种组合。

## 3. 工程视角

派生类型设计的关键不是“语法能不能写出来”，而是“接口是否清晰、生命周期是否一致、布局是否可预测”。后续小节会分别讨论每一种派生类型的声明规则、初始化方式和常见陷阱。
