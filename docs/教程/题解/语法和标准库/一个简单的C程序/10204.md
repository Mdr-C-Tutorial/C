# 10204

## 题目

触发断言：理解 `assert`。

1. 运行原始程序，并在第二个整数输入 `0`。
2. 仔细阅读并尝试理解控制台输出的错误信息。它告诉你错误发生在哪一个文件的哪一行吗？它告诉你失败的条件是什么吗？

## 解析

`assert(b != 0);` 表示程序作者认为执行到这里时，`b != 0` 应该成立。如果输入让 `b` 等于 0，这个条件失败，程序会中止并输出诊断信息。

## 答案

不同实现的输出格式可能不同，但通常会包含：

1. 失败的条件：`b != 0`；
2. 发生失败的源文件名；
3. 发生失败的行号；
4. 当前函数名。

因此，`assert` 适合用来暴露“程序本不应该走到这里”的错误。
