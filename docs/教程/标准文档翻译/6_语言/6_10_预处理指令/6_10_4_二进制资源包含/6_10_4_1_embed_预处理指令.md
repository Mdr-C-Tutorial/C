# 6.10.4.1 `#embed` 预处理指令

## 说明

1. 资源是翻译环境能够访问的数据来源。
2. 嵌入参数是嵌入参数序列中的单个预处理器参数。
3. 它有一个实现资源宽度，也就是被定位资源的实现定义位大小。
4. 它还有一个资源宽度，其值取以下两者之一：

- 如果提供了可选的 `limit` 嵌入参数，则取按该参数计算出的位数；
- 否则，取实现资源宽度。

5. 嵌入参数序列是一个以空白分隔的预处理器参数列表，这些参数可以修改 `#embed` 预处理指令替换结果。

## 约束

1. `#embed` 指令必须在给定嵌入参数的前提下，标识一个实现可以当作二进制数据序列处理的资源。
2. 本标准未规定的嵌入参数是实现定义的。实现定义的嵌入参数可以改变本小节后续定义的指令语义；否则，不含实现定义嵌入参数的 `#embed` 指令必须按本标准描述的方式工作。
3. 当资源宽度为 `0` 时，该资源视为空。
4. 设嵌入元素宽度为下列两者之一：

- 某个由实现定义嵌入参数确定的、大于 `0` 的整数常量表达式；
- `CHAR_BIT`。

5. `(资源宽度) % (嵌入元素宽度)` 的结果必须为 `0`。

## 语义

1. `#embed` 指令的展开结果，是由本小节后文描述的一串整数常量表达式组成的记号序列。
2. 列表中每个整数常量表达式对应的一组记号，都会用一个逗号与前一个表达式的记号组分隔。
3. 这个序列既不以逗号开始，也不以逗号结束。
4. 如果整数常量表达式列表为空，那么该记号序列也是空的。
5. 这条指令会被其展开结果替换；在某些嵌入参数存在时，还可能带有额外或替代的记号序列。

6. 形如

```c
#embed <h-char-sequence> embed-parameter-sequenceopt
```

的预处理指令，会在实现定义的一组位置中搜索由 `<` 与 `>` 之间序列唯一标识的资源。这个资源的搜索方式是实现定义的。

7. 形如

```c
#embed "q-char-sequence" embed-parameter-sequenceopt
```

的预处理指令，会在实现定义的一组位置中搜索由 `"` 分隔符之间序列唯一标识的资源。这个资源的搜索方式是实现定义的。如果实现不支持这种搜索，或者搜索失败，那么该指令会按如下形式重新处理：

```c
#embed <h-char-sequence> embed-parameter-sequenceopt
```

其中内部的 `q-char-sequence` 与原指令完全一致，包括其中可能出现的 `>` 字符。

8. 前述两种 `#embed` 形式，都按本小节后续规则执行。
9. 展开序列中各整数常量表达式的值，由实现对资源数据执行实现定义映射后得到。
10. 每个整数常量表达式的值都处于 `0` 到 `(2^(嵌入元素宽度)) - 1` 的闭区间内。
11. 如果同时满足以下条件：

- 这些整数常量表达式列表被用来初始化一个与 `unsigned char` 兼容的数组类型，或者在 `char` 不能表示负值时，用来初始化与 `char` 兼容的数组类型；
- 嵌入元素宽度等于 `CHAR_BIT`；

那么该数组各已初始化元素的内容，应当与“在翻译时把该资源的二进制数据用 `fread` 读入数组”这一效果一致。

12. 还允许使用形如

```c
#embed pp-tokens
```

但不匹配前述两种形式的预处理指令。此时，`embed` 后面的预处理记号会像普通文本那样处理。当前被定义为宏名的每个标识符，都会替换为其替换列表。所有替换完成后，得到的指令必须匹配前述两种形式之一。位于一对 `<` 与 `>` 预处理记号之间、或者一对 `"` 字符之间的预处理记号序列，如何合成为单个资源名预处理记号，是实现定义的。

13. 预处理器参数记号属于下列之一的嵌入参数，是标准嵌入参数：

```text
limit    prefix    suffix    if_empty
```

这些标准嵌入参数的含义在本小节后续部分规定。

## 推荐实践

1. `#embed` 指令的目标，是在可能的情况下，把资源中的二进制数据翻译成一串整数常量表达式，同时保留该资源位流的值。
2. 鼓励实现提供一种与源文件包含 `6.10.3` 的搜索路径类似、但彼此独立的机制。
3. 实现应同时考虑翻译时与执行时的位序和字节序，从而更准确地表示 `#embed` 指令引用资源的二进制数据。
4. 这样做可以尽量提高如下情况的一致性：若翻译时通过 `#embed` 引用的资源，与执行时通过常规方式访问到的是同一资源，那么例如用 `fread` 读入连续存储的数据，应尽可能与由 `#embed` 展开结果初始化得到的字符类型数组逐位一致。

## 示例

### 示例 1

放置一个较小的图像资源：

```c
#include <stddef.h>

void have_you_any_wool(const unsigned char*, size_t);

int main (int, char*[]) {
    static const unsigned char baa_baa[] = {
#embed "black_sheep.ico"
    };

    have_you_any_wool(baa_baa, sizeof(baa_baa));

    return 0;
}
```

### 示例 2

```c
int main (int, char*[]) {
    static const unsigned char coefficients[] = {
#embed "only_8_bits.bin" // 可能违反约束
    };

    return 0;
}
```

这段代码可能违反 `(资源宽度) % (嵌入元素宽度) == 0` 的约束。比如在 `CHAR_BIT == 16` 的系统上，这 `8` 位数据未必能被嵌入元素宽度整除。在这种情况下给出诊断，有助于提高可移植性。

### 示例 3

对非数组类型做初始化：

```c
int main () {
    /* 按通常的初始化规则，大括号可保留也可省略 */
    int i = {
#embed "i.dat"
    }; /* 当 i.dat 产生 1 个值时有效 */

    int i2 =
#embed "i.dat"
    ; /* 当 i.dat 产生 1 个值时有效 */

    struct s {
        double a, b, c;
        struct { double e, f, g; };
        double h, i, j;
    };

    struct s x = {
        /* 按初始化规则，用逗号分隔的整数常量表达式顺次初始化各元素 */
#embed "s.dat"
    };
    return 0;
}
```

非数组类型同样可以初始化，因为该指令可能产生一串以逗号分隔的整数常量表达式、单个整数常量表达式，或者空序列。

### 示例 4

同一资源在翻译时读取与执行时读取时，其位序列和位顺序的一致性：

```c
#include <string.h>
#include <stddef.h>
#include <stdio.h>

int main(void) {
    static const unsigned char embed_data[] = {
#embed <data.dat>
    };

    const size_t f_size = sizeof(embed_data);
    unsigned char f_data[f_size];
    FILE* f_source = fopen("data.dat", "rb");
    if (f_source == nullptr)
        return 1;
    char* f_ptr = (char*)&f_data[0];
    if (fread(f_ptr, 1, f_size, f_source) != f_size) {
        fclose(f_source);
        return 1;
    }
    fclose(f_source);

    int is_same = memcmp(&embed_data[0], f_ptr, f_size);
    // 如果翻译时和执行时访问的是同一资源/文件，那么 is_same 应当为 0
    return is_same == 0 ? 0 : 1;
}
```
