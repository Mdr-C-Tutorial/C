# 6.10.5.5 宏定义的作用域

## 语义

1. 宏定义的持续时间与块结构无关；它会一直持续到遇到对应的 `#undef` 指令为止，如果始终没有遇到，就持续到预处理翻译单元结束。
2. 宏定义在翻译阶段 `4` 之后不再有意义。
3. 形如

```c
#undef identifier
```

的预处理指令，会使指定标识符不再被定义为宏名。4. 如果该标识符当前并不是宏名，那么这条 `#undef` 会被忽略。

## 示例

### 示例 1

定义“清单常量”的最简单做法如下：

```c
#define TABSIZE 100

int table[TABSIZE];
```

### 示例 2

下面这个函数式宏给出两个实参中的较大值。它的优点是对任意兼容类型都能工作，并且能生成内联代码，不需要函数调用开销；缺点是其中一个实参可能会被再次求值，因此副作用也可能重复发生，而且多次使用时生成的代码可能比真正的函数更多：

```c
#define max(a, b) ((a) > (b) ? (a) : (b))
```

这些括号用于保证实参和整个结果表达式的结合方式正确。

### 示例 3

下面的序列：

```c
#define x      3
#define f(a)   f(x * (a))
#undef x
#define x      2
#define g      f
#define z      z[0]
#define h      g(~
#define m(a)   a(w)
#define w      0,1
#define t(a)   a
#define p()    int
#define q(x)   x
#define r(x,y) x ## y
#define str(x) # x

f(y+1) + f(f(z)) % t(t(g)(0) + t)(1);
g(x+(3,4)-w) | h 5) & m
      (f)^m(m);
p() i[q()] = { q(1), r(2,3), r(4,), r(,5), r(,) };
char c[2][6] = { str(hello), str() };
```

会得到：

```c
f(2 * (y+1)) + f(2 * (f(2 * (z[0])))) % f(2 * (0)) + t(1);
f(2 * (2+(3,4)-0,1)) | f(2 * (~ 5)) & f(2 * (0,1))^m(0,1);
int i[] = { 1, 23, 4, 5, };
char c[2][6] = { "hello", "" };
```
