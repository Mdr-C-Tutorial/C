# 6.10.5.1 实参替换

## 语法

::: details 形式化语法

```bnf
va-opt-replacement:
    __VA_OPT__ ( pp-tokensopt )
```

:::

## 说明

1. 实参替换是在宏展开期间发生的一个过程。
2. 在这个过程中，宏定义里对应形参的标识符，以及特殊构造 `__VA_ARGS__` 与 `__VA_OPT__`，都会被宏调用实参提供的记号序列替换。
3. `__VA_OPT__` 还允许控制一个只在尾随 `...` 形参对应的实参存在且替换结果非空时才会展开的替代记号序列。

## 约束

1. 标识符 `__VA_OPT__` 必须始终以 `va-opt-replacement` 的形式出现。
2. 它的结束 `)`，通过跳过其中内部配对的圆括号来确定。
3. `va-opt-replacement` 内部的 `pp-tokens` 不得再包含 `__VA_OPT__`。
4. 这些 `pp-tokens` 必须能够构成当前函数式宏的一个有效替换列表。

## 语义

1. 当函数式宏调用的实参都被识别出来之后，就开始做实参替换。
2. `va-opt-replacement` 视作一种特殊形参。
3. 对替换列表中的每个形参，只要它既不紧跟在 `#` 或 `##` 之后，也不紧跟在 `##` 之前，就用对应实参确定的记号序列替换它。
4. 如果该形参本身是 `va-opt-replacement` 形式，那么替换记号序列由本小节后文规定。
5. 否则，替换记号序列就是对应实参中所有宏都展开之后得到的预处理记号序列；其展开方式如同这些记号单独构成了预处理文件剩余部分。
6. 出现在替换列表里的 `__VA_ARGS__` 视作形参处理，并由可变实参构成的预处理记号序列替换。
7. 对 `va-opt-replacement` 来说，如果把 `__VA_ARGS__` 假想地按既不是 `#` 也不是 `##` 的操作数来替换后，得到的是空序列，那么对应实参视为只包含一个占位预处理记号。
8. 否则，该实参由其中 `pp-tokens` 按当前函数式宏替换列表先展开后得到的结果构成；这里是在移除占位记号、重新扫描和进一步替换之前的阶段。

## 示例

### 示例 1

```c
#define LPAREN() (
#define G(Q) 42
#define F(R, X, ...) __VA_OPT__(G R X) )
int x = F(LPAREN(), 0, <:-);    // 替换后得到 int x = 42;
```

### 示例 2

```c
#define F(...)           f(0 __VA_OPT__(,) __VA_ARGS__)
#define G(X, ...)        f(0, X __VA_OPT__(,) __VA_ARGS__)
#define SDEF(sname, ...) S sname __VA_OPT__(= { __VA_ARGS__ })
#define EMP
```
