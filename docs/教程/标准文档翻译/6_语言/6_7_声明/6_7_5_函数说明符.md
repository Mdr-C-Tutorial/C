# 6.7.5 函数说明符

[[toc]]

## 可用说明符

1. 函数说明符只有两个：
   - `inline`
   - `_Noreturn`

## 约束

1. 函数说明符只能用于函数标识符的声明。

2. 具有外部链接的 `inline` 函数定义，其函数体内不得：
   - 定义可修改的静态存储期或线程存储期对象；
   - 引用具有内部链接的标识符。

3. 在宿主环境中，`main` 的声明里不得出现函数说明符。

## 语义

1. 函数说明符可以重复写；效果与只写一次相同。

2. `inline` 表示该函数是内联函数，含义是“建议实现尽量让调用更快”，但是否真的做内联替换，由实现定义。

3. 任何具有内部链接的函数都可以是内联函数。

4. 对具有外部链接的函数，如果声明里出现了 `inline`，那么同一翻译单元里还必须给出该函数的定义。

5. 若某翻译单元中该函数的所有文件作用域声明都带 `inline` 且不带 `extern`，则该翻译单元中的定义是**内联定义**。

6. 内联定义本身不提供外部定义，也不阻止别的翻译单元再提供一个外部定义。

7. 对同一调用而言，最终用的是当前翻译单元里的内联定义，还是别处的外部定义，标准规定为未指定。

8. `_Noreturn` 表示函数不应返回给调用者。

9. `[[noreturn]]` 属性提供类似语义；`_Noreturn` 作为函数说明符已经属于将来可能移除的旧特性。

## 示例理解

1. 下面这个翻译单元里，`fahr` 因为又有一条 `extern` 声明，所以形成外部定义；`cels` 则只是内联定义：

```c
inline double fahr(double t) { return (9.0 * t) / 5.0 + 32.0; }
inline double cels(double t) { return (5.0 * (t - 32.0)) / 9.0; }
extern double fahr(double);
```

2. 实现应当对“看起来可能返回”的 `_Noreturn` 函数给出诊断。
