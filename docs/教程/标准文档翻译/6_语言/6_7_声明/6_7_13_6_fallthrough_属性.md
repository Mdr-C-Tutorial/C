# 6.7.13.6 fallthrough 属性

[[toc]]

## 约束

1. `fallthrough` 只能出现在**属性声明**中，也就是类似：

```c
[[fallthrough]];
```

2. 它不能带属性参数子句。

3. 它只能出现在某个外围 `switch` 语句内部。

4. 在它之后按控制流将要遇到的下一个块项，必须是最内层 `switch` 对应的 `case` 或 `default` 标号。

5. 若它又位于某个迭代语句内部，那么那个“下一个语句”还必须属于该最内层迭代语句同一次次级块执行。

## 语义

1. 若实现支持该属性，则：

```c
__has_c_attribute(fallthrough)
```

应返回 `202311L`。

## 推荐实践

1. `fallthrough` 的目的，是压制实现原本可能给出的“从一个 `case` 落入下一个 `case`”诊断。

2. 如果 `[[fallthrough]];` 这条声明在运行时根本不可达，实现反而应倾向于给出诊断。

3. 放在循环里但下一条并不属于同一次循环执行的那类写法，标准示例明确判为约束违规。
